Privacy Policy

Effective date: March 8, 2026

Local List Assist is designed around local-first software. The Android app stores list data on the device by default. The website at locallistassist.app exists to provide product information, support contact details, and this public privacy policy.

Android app data

The current Android app does not require an account and does not depend on a remote backend to function.

  • List names, colors, categories, and ordering are stored locally on device.
  • Items, completed history, archived lists, and recent activity are stored locally on device.
  • Backups created through the app are stored in app-managed device storage.
  • The current version does not include ads, third-party analytics, or account-based tracking.

Website data

The website is a simple public marketing and policy site. If standard web hosting logs are collected by the hosting provider, they are limited to ordinary operational data such as IP address, request path, and user agent for delivery and security purposes.

Data sharing

Local List Assist does not sell personal data. The current Android app does not transmit your list contents to a Local List Assist server.

Backups

If you export a backup from the Android app, the resulting file is stored on the device. You control whether it is retained, copied elsewhere, or deleted.

Home Assistant version

The free Home Assistant integration runs inside Home Assistant and stores its data in the user’s HA environment. That version is separate from the standalone Android app and follows the behavior of the user’s Home Assistant installation and hosting setup.

Future changes

If future versions add optional features such as notifications, widgets, sharing, cloud sync, or voice input, this policy will be updated to reflect those changes before broad rollout.

Contact

Questions about privacy or support can be sent to support@locallistassist.app.
